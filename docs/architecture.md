# Architecture

## Routes

Expo Router owns navigation:

- `/` dashboard
- `/statistics` collection statistics
- `/records` catalog
- `/records/[releaseId]` detail
- `/statistics/breakdowns/[dimension]` full stats breakdowns

Route files stay thin and delegate to feature components under `src/features`.
Dashboard, statistics, and records each own a nested stack inside the tab navigator so subpages keep the bottom tab bar visible while still getting their own headers.

## API Layer

The API contract is generated by `openapi-typescript` into:

```bash
src/api/generated/record-collection-api.ts
```

The runtime client is hand-written in `src/api/client.ts` to keep dependencies small. It handles:

- Base URL normalization
- Optional `x-api-key`
- Query-string serialization
- Request timeout
- Safe API error normalization

React Query hooks in `src/api/queries.ts` handle caching, pagination, retries, and loading states.

## Localization

The app uses a small typed localization layer in `src/localization/`. Finnish is the active locale today, but all user-facing copy flows through translation keys so more locales can be added without rewriting screens or tests.

## UI Composition

UI reuse follows a small-step approach:

- Keep one-off layout and spacing inline in the owning screen or component.
- Move repeated visual recipes into shared theme styles such as screen containers, framed panels, wrap rows, and chip bases.
- Promote a pattern to a shared component only when it includes styling plus behavior, interaction state, or accessibility semantics.

This keeps Expo UI code direct to read while still giving repeated interactive patterns, such as chips or panels, a single implementation.

## Runtime Configuration

API URL and required read API key come from Expo/EAS environment variables. `app.config.js` maps `API_URL` and `API_KEY` into Expo config so the client can read them at runtime. The app also accepts `EXPO_PUBLIC_API_URL` and `EXPO_PUBLIC_API_KEY` for local public Expo env workflows.

React Query keys include the API URL, but never the API key value.
